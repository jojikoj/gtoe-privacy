AI導入支援 ── 3プラン戦略

株式会社TOE / 社内用 / 2026.06
プランを分ける軸は2本。①助言だけか / TOEが実装するか、 ②Claude Codeを顧客環境に入れるか(=Standard / Premium の決定的な境界)。 この一線で「データの所在・提供価値・責任範囲」が同時に切り替わる。 3プランはフロー(売り切り)、その先に継続契約(ストック)を置く。

3プラン全体像

　① Light(診断・伴走)② Standard(実装代行)③ Premium(内製化)
ひとこと何にAIを使うか決めるTOEが作って渡す自走する仕組みを社内に残す
Claude Code入れないTOE側で動かす顧客環境に入れる
提供物業務棚卸し・導入ロードマップ動く自動化アプリ＋活用研修環境構築＋運用設計＋構築研修
価値の型意思決定支援(フロー)成果物(フロー)内製能力(ストック)
課金低・スポット中・プロジェクト高・構築＋継続

Standard をどこまでやるか(設計の肝)

やる

  • ✓ 痛い業務の自動化アプリ構築(動けばいい・作り込み不要)
  • ✓ 作ったアプリの活用研修
  • ✓ 納品後 2〜4週間のバグ修正(動作保証)

やらない(＝次の発注 or Premium)

  • ✕ 業務ロジックの変更・新機能追加
  • ✕ 自分たちで作れるよう教える(＝Premiumの中身)
“育てたい”欲求が出た瞬間に天井に当たり、Premiumへ。

補助金戦略(人材開発支援助成金)

① 請求を2本立てにする

  • 構築費 … 対象外(顧客負担 / IT導入補助金など別枠)
  • 研修費 … 助成金を乗せる
アプリ作りを“研修”に偽装しない。別建ての本物の研修を付ける。

② 研修を“本物のリスキリング”に

  • 10時間以上の汎用AI活用カリキュラム、作ったアプリは実習教材
  • 操作説明はNG → 応用の効くスキルに
  • 事前申請必須 / 社労士と提携し事務を巻き取る

Premium の後 ── 継続課金(ストック層)

月額顧問 / リテーナー

  • 最新追従(AIの進化が速い＝継続価値の本命)
  • 新部署・新業務への横展開支援(また補助金)
  • 利用ガバナンス・コスト・品質管理

鉄則

  • Premium単体で黒字にする
  • 後続(顧問)は当てにしない＝取れたらボーナス
  • 自立して離れる顧客は必ず出る前提で設計
Light(診断) → Standard(実装＋活用研修) → Premium(内製化＋構築研修) → 継続顧問(ストック)
