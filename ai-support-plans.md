# AI導入支援 3プラン戦略

最終更新: 2026年6月 / 株式会社TOE

## コンセプト

AI導入支援サービスを、顧客の成熟度に沿った3段階の階段として設計する。
プランを分ける軸は2本の境界線。

- **境界①**: 手を動かすのは誰か(助言だけ vs TOEが実装)
- **境界②**: Claude Code をクライアント環境に入れるか(成果物納品 vs 内製化)

特に **境界②(Claude Code をクライアントに入れるか)が Standard と Premium の決定的な境界**。
この一線で「データの所在」「提供価値(フロー/ストック)」「責任範囲」が同時に切り替わり、
単なる機能差ではなくビジネスモデルそのものが分岐する。

## 3プラン一覧

| | ① Light(診断・伴走) | ② Standard(実装代行) | ③ Premium(内製化) |
|---|---|---|---|
| ひとことで | 「何にAIを使うか」を決める | 「TOEが作って渡す」 | 「自走する仕組みを社内に残す」 |
| Claude Code | 入れない | TOE側で動かす | クライアント環境に入れる |
| 主な提供物 | 業務棚卸し・導入ロードマップ・PoC方針 | 成果物(自動化・ツール・ドキュメント) | 環境構築 + 運用設計 + 社内トレーニング |
| データの所在 | クライアント側(見るだけ) | TOE側で完結 | クライアント環境に踏み込む |
| 価値の型 | 意思決定支援(フロー) | 成果物(フロー) | 内製能力(ストック) |
| 責任範囲 | 助言の質 | 納品物の品質 | 環境・セキュリティ・運用継続 |
| 課金イメージ | 低・スポット/月額小 | 中・プロジェクト単位 | 高・初期構築 + 継続運用 |

## このストーリーの強み

3プランが機能の足し算ではなく、顧客の成熟度に沿った階段になっている。

1. **Light** で信頼と現状把握を得る → Standard への自然な入口
2. **Standard** で成果を体感させる → 「自社でも回したい」が生まれる
3. **Premium** で内製化 → 顧客に資産が残り、TOEは継続収益(運用・保守)

各段で TOE のリスクと責任が一段ずつ上がるため、価格を大きく分ける根拠が明確。

## 今後詰める論点

1. **価格設定** — 各プランの金額レンジと課金方式(スポット/月額/成果報酬)
2. **Premiumの責任設計** — クライアント環境に入る以上、セキュリティ・情報管理・契約条項をどう守るか(最重要リスク)
3. **顧客像とアップセル動線** — どんな企業にどう売り、Light→Standard→Premium へどう引き上げるか
